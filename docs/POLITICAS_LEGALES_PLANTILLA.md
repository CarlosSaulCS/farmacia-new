# Politicas legales y aviso para clientes

Documento base para revision legal. No sustituye asesoria juridica. Antes de vender publicamente, Code Solutions Studio debe validar este texto con un abogado, especialmente porque el sistema puede registrar datos de pacientes y servicios de salud.

Empresa desarrolladora: Code Solutions Studio  
Contacto: contacto@codesolutionsstudio.com.mx  
Producto: Farmacia

## 1. Alcance del software

Farmacia es un sistema local de administracion para:

- punto de venta
- inventario
- citas
- servicios
- caja
- reportes
- respaldos
- alertas operativas
- analisis local e historico

El sistema no sustituye el criterio medico, farmaceutico, contable, fiscal o legal del usuario.

## 2. Uso en salud

El sistema puede capturar datos relacionados con pacientes, citas, servicios y seguimientos. Estos datos son responsabilidad del cliente que opera la aplicacion.

No debe promocionarse como expediente clinico electronico certificado ni como dispositivo medico. Si el cliente desea usarlo como expediente clinico formal, debe validar cumplimiento normativo aplicable con asesor juridico y sanitario.

## 3. Responsabilidad sobre datos

La aplicacion funciona principalmente de forma local. La base de datos se almacena en el equipo del cliente.

El cliente es responsable de:

- obtener consentimientos necesarios
- informar a pacientes y usuarios sobre el tratamiento de datos
- proteger el equipo con contrasena y antivirus
- limitar accesos internos
- realizar respaldos
- conservar o eliminar datos conforme a sus obligaciones

Code Solutions Studio no accede a la base de datos local salvo que el cliente entregue voluntariamente un respaldo para soporte.

## 4. Datos personales y datos sensibles

El sistema puede contener datos personales y datos relacionados con salud. En Mexico, los datos de salud pueden considerarse sensibles y requieren cuidado reforzado.

El cliente debe contar con su propio aviso de privacidad y procedimientos internos de proteccion de datos.

## 5. Uso de IA o servicios externos

El sistema puede funcionar con reglas locales sin IA. Si se configura una API externa de IA, el cliente debe revisar que datos se envian y obtener autorizaciones necesarias.

Recomendacion comercial: no enviar datos personales identificables a servicios externos de IA salvo que exista base legal, contrato, consentimiento y configuracion adecuada.

Code Solutions Studio concede al cliente una licencia de uso del software, no una transferencia de propiedad intelectual.

Queda prohibido:

- revender el software sin autorizacion
- copiarlo para terceros
- modificarlo o descompilarlo sin permiso
- remover marcas o avisos de propiedad
- usarlo para actividades ilicitas

## 8. Respaldos y recuperacion

El cliente debe generar respaldos periodicos desde el sistema y conservarlos en un medio seguro.

Code Solutions Studio no garantiza recuperacion de informacion si:

- el cliente no genero respaldos
- el equipo fue danado
- el archivo de base de datos fue borrado
- hubo malware, formateo o perdida del dispositivo

## 9. Limitacion de responsabilidad

El software se entrega como herramienta administrativa. Code Solutions Studio no sera responsable por:

- errores derivados de captura incorrecta
- decisiones medicas, comerciales, contables o fiscales tomadas por el cliente
- perdida de datos cuando no existan respaldos
- fallas del equipo del cliente
- uso fuera del alcance contratado

## 10. Actualizaciones

Las actualizaciones pueden incluir mejoras, correcciones y cambios de interfaz. Antes de actualizar, el cliente debe generar respaldo.

Code Solutions Studio debe conservar historial de cambios por version.

Farmacia opera localmente en el equipo del cliente. El cliente conserva el control de sus datos y es responsable de respaldos, privacidad y uso correcto del sistema. Code Solutions Studio proporciona el software, soporte y actualizaciones conforme al plan contratado.
